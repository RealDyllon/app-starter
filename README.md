# app-starter

Monorepo starter centered on a TanStack Start web app in `apps/web`.

## What is here

- `apps/web`: React 19 + TanStack Start + Vite + Nitro
- `apps/web/src/routes`: file-based routes, including `/`, `/about`, `/todos`, `/login`, and `/signup`
- `apps/web/src/routes/api*`: colocated API handlers for OpenAPI docs, oRPC, and Better Auth
- `apps/web/src/server/db`: Drizzle schema and Postgres database wiring
- `apps/web/src/server/orpc`: typed procedure router and server handlers

## Stack

- TanStack Start + TanStack Router
- TanStack Query + router SSR query integration
- oRPC for typed API calls
- Better Auth for email/password auth
- Drizzle ORM + PostgreSQL
- Paraglide for i18n
- Biome for formatting and linting

## Requirements

- Node.js 24+
- pnpm 10+
- PostgreSQL for the persistent todo slice and Better Auth storage

## Install

From the repo root:

```bash
pnpm install
```

## Environment

Copy `apps/web/.env.example` to `apps/web/.env.local` and set:

```env
DATABASE_URL="postgresql://username:password@localhost:5432/app_starter"
BETTER_AUTH_URL="http://localhost:3000"
BETTER_AUTH_SECRET="replace-with-a-generated-secret"
SERVER_URL="http://localhost:3000"
VITE_APP_TITLE="app-starter"
```

Set either `BETTER_AUTH_URL` or `SERVER_URL` for Better Auth. If both are present, `BETTER_AUTH_URL` wins and `SERVER_URL` remains a trusted origin fallback.

## Daily commands

From the repo root:

```bash
pnpm web:dev
pnpm build
pnpm test
pnpm check
pnpm typecheck
```

The app runs on [http://localhost:3000](http://localhost:3000).

## Database and auth setup

Start local Postgres:

```bash
docker compose up -d postgres
```

Generate or refresh the app migration from `apps/web/src/server/db`:

```bash
cd apps/web
pnpm db:generate
```

Apply the Drizzle migrations:

```bash
pnpm db:migrate
```

Better Auth uses the same Postgres database connection. Its tables are exported from `apps/web/src/server/db/schema.ts` and managed by the Drizzle migrations.

Generate `BETTER_AUTH_SECRET` with a local secret generator such as `openssl rand -base64 32`.

## Docker

Build from the repo root so the Dockerfile can see the workspace files:

```bash
docker build -f apps/web/Dockerfile -t app-starter-web .
```

## Generated files

- `apps/web/src/routeTree.gen.ts` is generated by the TanStack tooling. Do not edit it by hand.
- `apps/web/src/i18n/paraglide/*` is generated by Paraglide during `pnpm dev` and `pnpm build`.

## Notes

- Keep `devtools()` first in `apps/web/vite.config.ts`.
- Keep `tanstackStart()` before `viteReact()` in `apps/web/vite.config.ts`.
- The `/todos` route is the reference vertical slice for routing, auth, data fetching, and persistence.
